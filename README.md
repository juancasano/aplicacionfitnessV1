# Aplicación Fitness - Documentación modular

Proyecto de ejemplo: aplicación CRUD modular para gestionar ejercicios, entrenamientos y usuarios.

Contenido principal:

- `spec.md` — Especificación general y reglas globales.
- `docs/database.md` — Fuente única de verdad para las tablas y relaciones.
- `docs/architecture.md` — Arquitectura técnica y estructura de carpetas.
- `docs/modules/` — Especificaciones e implementación por módulo.
- `app/` — Esqueleto de la aplicación (HTML/CSS/JS).

Demo publicada:

- [Abrir la app de ejercicios](https://juancasano.github.io/aplicacionfitnessV1/app/exercises.html)

Objetivo: servir como plantilla y punto de partida para desarrollar una app de fitness
con documentación generada por prompt y mantenida de forma modular.
